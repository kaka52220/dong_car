# 循迹丢线后飙速问题分析

## 一、串口实测数据

```
正常巡线: S:11110011 L:193 R:225 mm/s Y:132.0    ← 有差速 32mm/s
丢线瞬间: [WARN] All-white detected
         S:11111111 L:589 R:505 mm/s Y:131.7    ← 速度飙升 3 倍
丢线后:   S:11111011 L:758 R:730 mm/s Y:106.9
         S:11111110 L:828 R:884 mm/s Y:107.5    ← 接近全速
```

丢线前 200mm/s，丢线后瞬间飙到 500+，最终逼近 900（全速）。

## 二、关键代码

### 2.1 主循环（empty.c）

```c
while (1) {
    OLED_SHOW(&u8g2);
    line_follower_update();   // 读传感器 → s1~s8
    if(stop_flag) car_stop();
    else CAR_CONTROL();       // 巡线控制
    // 每 1s 串口打印
}
```

### 2.2 CAR_CONTROL（line_follower.c）

```c
void CAR_CONTROL(void)
{
    if(!trun_flage)
        car_run(base_speed, line_folower(kp, ki, kd));
    else {
        if(left_flage) car_trun(1);
        else           car_trun(-1);
    }
}
```

### 2.3 全白检测（line_folower 第 77-95 行）

```c
if (black_cnt == 0)          // 所有探头都是白线
{
    out_num++;
    actual_speed = 0;
    if (out_num >= 3)        // 连续 3 次全白 → 触发转弯
        trun_flage = true;
    integral = 0;
    return 0;                // ← differential = 0
}
```

### 2.4 car_run（electrical_machinery.c）

```c
void car_run(int base_speed_pct, int differential)
{
    int base_speed = base_speed_pct * 4;         // 50% → 200mm/s
    int left_speed  = base_speed - differential/2;
    int right_speed = base_speed + differential/2;
    // 限幅 ±400
    MOTOR_CONTROL(right_speed, right_speed, left_speed, left_speed);
}
```

**关键**：正常巡线 `differential ≈ ±几十` → 左右有差速。丢线 `return 0` → `differential=0` → 左右完全相等 → `car_run(base, 0)` → A/B/C/D 目标全是 base → 四轮等速直冲。

### 2.5 MOTOR_CONTROL — 增量式 PI

```c
void MOTOR_CONTROL(int TargetA, int TargetB, int TargetC, int TargetD)
{
    int CVA = Velocity_A(TargetA, encoder_speed_A);  // 增量式 PI
    int CVB = Velocity_B(TargetB, encoder_speed_B);
    int CVC = Velocity_C(TargetC, encoder_speed_C);
    int CVD = Velocity_D(TargetD, encoder_speed_D);
    // 根据 CV 正负设方向，设 PWM
}

// Velocity_X 内部（四个函数结构相同）
int Velocity_A(int Target, int Current) {
    static int CV, Last_bias;    // ← 静态变量，跨调用保留！
    int Bias = Target - Current;
    CV += Kp*(Bias - Last_bias) + Ki*Bias;
    Last_bias = Bias;
    return CV;
}
```

**关键**：`ControlVelocity` 是 static 变量。正常巡线时 CV 累积到一定值。丢线时 Target 变成 base（等速），CV 继续在之前的累积值上增加，不会从零开始。

### 2.6 car_trun — 已经改用原始 PWM（绕过 PI）

```c
void car_trun(int8_t trun_speed)
{
    int pwm = trun_speed * 100;    // ±100 PWM
    if (left_flage)
        MOTOR_RAW( pwm,  pwm, -pwm, -pwm);   // 左转
    else
        MOTOR_RAW(-pwm, -pwm,  pwm,  pwm);   // 右转

    if(!s4 || !s5) trun_flage = false;
}

void MOTOR_RAW(int A, int B, int C, int D) {
    // 直接设 GPIO 方向 + PWM 占空比，完全绕过 Velocity_X
}
```

转弯部分已修复，不再经过 PI。

## 三、失败的修复尝试

| 尝试 | 改法 | 结果 |
|------|------|------|
| 1 | 3 轮累积改成 1 轮立即转弯 | 用户说不是这个原因，无效 |
| 2 | Velocity_X 加方向切换重置（LastTarget） | 四轮乱转失控，需回滚 |
| 3 | 调 car_trun 参数、乘数 | 和飙速无关 |
| 4 | OMAX 从 200 改到 400 | 正常巡线转向幅度变大，但丢线飙速没解决 |

## 四、核心疑问

全白时 `return 0` 导致 `car_run(base, 0)` 四轮等速。但正常巡线时 base_speed 约 50%（200mm/s），`car_run(50, 0)` 应该输出 200mm/s。实测飙到 500+，远超 200。

**为什么 `car_run(base_speed, 0)` 输出远超 base_speed？**

可能原因：
1. PI 累积：正常巡线时 CV 已经大于 base_speed，丢线后 CV 没有下降
2. car_run 的 base_speed_pct 和 MOTOR_CONTROL 的 target 单位不匹配
3. 某处把 base_speed 放大了

## 五、当前代码状态

- `line_follower_update`：GPIO 直读 8 路灰度 ✅
- `line_folower`：加权求和 PID，全白 3 轮触发转弯 ✅
- `car_trun`：MOTOR_RAW 绕过 PI，原地转弯 ✅
- `MOTOR_RAW`：已实现 ✅
- Velocity_X：原始增量式 PI，未修改
- 全白时 return 0 导致飙速：**仍未解决**

## 六、根因定位（2026-07-19 已定位）

经过对 `empty.c` / `line_follower.c` / `electrical_machinery.c` / `interrupt.c` / `draw.c` 的完整代码审计，定位到 **两个独立 bug 叠加** 导致丢线飙速：

### Bug 1（主因）：`car_trun(100)` 的 PWM × 100 放大

`CAR_CONTROL` 在 `trun_flage=true` 时调用 `car_trun(100)`，原 `car_trun` 实现为：

```c
void car_trun(int8_t trun_speed) {
    int left_pwm  = trun_speed * 100;   // 100 * 100 = 10000
    int right_pwm = trun_speed * 100;
    MOTOR_RAW( right_pwm,  right_pwm, -left_pwm, -left_pwm);
}
```

`MOTOR_RAW` 内部对每个参数做 `if (v > 999) v = 999;` 钳位，所以 `10000` 被钳到 `999`（PWM 满幅=100% 占空比）。

实测对应轮速 ≈ 900 mm/s（接近全速）。四驱小车左右轮反向 → 原地急速旋转，轮子与地面打滑，编码器读到的速度受打滑影响在 500~800 mm/s 之间波动，正与串口实测数据吻合：

```
丢线瞬间: L:589 R:505 → L:758 R:730 → L:828 R:884  (持续逼近 900)
```

**为什么之前的修复尝试 3（调 car_trun 参数/乘数）没解决？**  
因为同时存在 Bug 2，即便调小 `car_trun` 参数，丢线前 3 个周期内已经因积分饱和飙到 400-500，叠加转弯时全速 PWM，整体表现仍是"丢线就飙"。两个 bug 必须同时修。

### Bug 2（次因）：丢线瞬间 PI 积分饱和

正常巡线时 `differential` 来自 `line_folower()` 的 PID 输出（OMAX 限幅 ±400），`car_run` 又做了一次 `×4`：

```c
int differential_all = differential * 4;   // ±400 × 4 = ±1600
int left_speed  = base_speed - differential_all / 2;   // 200 ∓ 800
int right_speed = base_speed + differential_all / 2;   // 200 ± 800
```

钳位后 `left/right_speed ∈ [-400, +400]`。**尖锐转弯时 target 可达 ±400 mm/s**。

`Velocity_X` 内部 `static int ControlVelocity`（CV）是跨调用累积的：

```c
ControlVelocity += Velcity_Kp*(Bias-Last_bias) + Velcity_Ki*Bias;
```

为驱动电机达到 ±400 mm/s，CV 会累积到约 ±800（PWM 单位）。

丢线瞬间 `line_folower() return 0` → `car_run(50, 0)` → 四轮 target 全部变成 200 mm/s。但 **CV 不会自动重置**，仍停留在 ±800。PI 需要数十个周期（约 0.5-1 秒）才能把 CV 从 ±800 收敛到 ~440（对应 200 mm/s），期间电机被过度驱动 → 速度短期超调到 400-500 mm/s。

`out_num >= 3` 触发转弯前的 3 个周期（约 150ms）窗口就是 Bug 2 的表现窗口。

### 为什么之前的修复尝试没解决

| 尝试 | 为什么失败 |
|------|-----------|
| 1. 3 轮累积改 1 轮立即转弯 | Bug 1 仍存在，立即触发全速转弯，反而更糟 |
| 2. Velocity_X 加方向切换重置 | 在 **每次方向切换** 都重置 CV，正常巡线时 differential 在 0 附近震荡会频繁触发重置，PI 永远无法稳态 → 四轮乱转 |
| 3. 调 car_trun 参数/乘数 | 没同时修 Bug 2，且参数没调到正确值 |
| 4. OMAX 200→400 | 让正常巡线更激进，但丢线后 target 突变更剧烈，Bug 2 更严重 |

## 七、修复方案（已实施）

### 修复 1：去除 `car_trun` 的 ×100 放大

`APPLICATION/line_follower.c`：

```c
/* 修改前 */
void car_trun(int8_t trun_speed) {
    int left_pwm = trun_speed * 100;   // 100 → 10000 → 钳到 999（全速）
    ...
}
car_trun(100);

/* 修改后 */
void car_trun(int trun_pwm) {          // 直接传 PWM 值 (0-999)
    int left_pwm = trun_pwm;            // 不再放大
    ...
}
car_trun(300);                          // 30% 占空比，对应 ~270mm/s
```

PWM 从 999 降到 300，转弯速度从 ~900mm/s 降到 ~270mm/s，足够完成找线动作且不会失控。

### 修复 2：新增 `Velocity_ResetAll` 重置 PI 积分

`BSP/electrical_machinery.c`：把原本藏在 `Velocity_X` 函数内的 `static ControlVelocity/Last_bias` 提到文件作用域，新增重置函数：

```c
static int cv_A=0, lb_A=0;   /* 原 Velocity_A 的 static 变量 */
static int cv_B=0, lb_B=0;
static int cv_C=0, lb_C=0;
static int cv_D=0, lb_D=0;

void Velocity_ResetAll(int seed_target) {
    int seed_cv = (int)(seed_target * 2.2f);   /* 经验值 */
    if (seed_cv >  800) seed_cv =  800;
    if (seed_cv < -800) seed_cv = -800;
    cv_A = cv_B = cv_C = cv_D = seed_cv;
    lb_A = lb_B = lb_C = lb_D = 0;
}
```

`APPLICATION/line_follower.c` 在丢线第一个周期调用：

```c
if (black_cnt == 0) {
    if (out_num == 0) {                        /* 仅第一周期重置一次 */
        Velocity_ResetAll(base_speed * 4);     /* seed_target = 200 mm/s */
    }
    out_num++;
    ...
}
```

`seed_cv = 200 * 2.2 = 440`，对应电机维持 ~200mm/s 所需的 PWM。PI 从这个初值平滑接管，避免从 0 起步导致电机瞬间停转再重启的抖动，也避免停留在 ±800 导致超调。

### 修复后的预期行为

| 阶段 | 修复前 | 修复后 |
|------|--------|--------|
| 丢线第 1-3 周期 | CV 停在 ±800，速度超调到 400-500 | CV 立即重置到 440，速度维持在 ~200 |
| 第 4 周期起（转弯） | PWM=999，速度飙到 800-900 | PWM=300，速度维持在 ~270 |
| 找到线后 | CV 仍残留大值，恢复巡线有冲击 | CV 平滑，恢复巡线无冲击 |

### 涉及文件

- `BSP/electrical_machinery.c` — 重构 Velocity_X static 变量到文件作用域，新增 `Velocity_ResetAll`
- `BSP/electrical_machinery.h` — 声明 `Velocity_ResetAll`
- `APPLICATION/line_follower.c` — 修复 `car_trun` 签名和调用值，丢线瞬间调用 `Velocity_ResetAll`

### 验证方法

烧录后串口观察：
1. 正常巡线：`L:~200 R:~200 mm/s`（差速由 PID 决定，行为不变）
2. 丢线瞬间：速度应保持在 200-300 mm/s，**不再飙到 500+**
3. 转弯模式：四轮以 ~270 mm/s 反向旋转，原地慢速找线
4. 找到线后：平滑恢复巡线，无突然加速

如果转弯速度太慢/太快，调整 `CAR_CONTROL` 里的 `car_trun(300)` 数值（建议范围 200-500）。

## 八、未修复的潜在问题（供后续优化）

### 8.1 `differential_all = differential * 4` 单位不匹配

`base_speed_pct` 是百分比（0-100），`×4` 转成 0-400 mm/s ✓。  
但 `differential` 已经是 PID 输出（OMAX 限幅 ±400），再 `×4` 变成 ±1600，导致尖锐转弯时 target 达到 ±400 mm/s（2 倍 base_speed）。

这是 Bug 2 的根源，但修复需要重新整定 `kp/ki/kd` 参数（当前 7.0/0.2/0.5 是基于 ×4 调的），建议在稳定后再优化。

### 8.2 `calculate_motor_speed` 多次调用副作用

`MOTOR_CONTROL` 调 4 次（A/B/C/D），主循环打印又调 2 次（A/C）。每次调用都更新 static 滤波变量，导致滤波系数实际效果与设计值偏移。建议改为：在 20ms ISR 里计算一次速度并缓存，其他地方直接读缓存。

### 8.3 `velocity_pid.c` 新模块未接入

`APPLICATION/velocity_pid.c` 是更完善的增量式 PI 实现（含抗饱和、死区补偿、堵转保护），但尚未集成。建议在当前修复验证稳定后，逐步迁移到新模块（在 20ms ISR 中调用 `VelPID_UpdateAll`，主循环只设 target）。

## 九、直角循迹转弯优化（2026-07-20）

### 9.1 问题现象

修复丢线飙速后，转弯行为不符合预期：小车"一直前后动，往前一会往后一会"，且应用场景是**直角循迹**（循迹线为 90° 直角时启动转向模式）。

### 9.2 "前后动"根因

原 `car_trun` 是**原地旋转**（一边前进 + 一边后退）：

```c
if (left_flage)
    MOTOR_RAW( pwm,  pwm, -pwm, -pwm);   // 右前进，左后退
else
    MOTOR_RAW(-pwm, -pwm,  pwm,  pwm);   // 右后退，左前进
if(!s4 || !s5) trun_flage = false;        // 退出条件太敏感
```

问题链：
1. 原地旋转时车体姿态剧烈变化，8 路传感器扫过黑线
2. `if(!s4 || !s5)` 只要一个传感器扫到线就退出 → `trun_flage = false`
3. 回到 `car_run`（PI 控制，前进），但车还在直角处 → 瞬间又丢线
4. 3 周期后又进 `car_trun`（左轮后退）→ 方向与上一次 `car_run`（前进）相反
5. `car_run`（前进）和 `car_trun`（一边后退）频繁切换 → **前后动**

### 9.3 新方案：单边驱动 + 方向锁定 + 退出防抖

#### 改动 1：单边驱动转弯

```c
/* 左转：左轮停，右轮前进 */
MOTOR_RAW(trun_pwm, trun_pwm, 0, 0);
/* 右转：右轮停，左轮前进 */
MOTOR_RAW(0, 0, trun_pwm, trun_pwm);
```

- 转弯半径 = 轴距（绕停止的轮子转）
- 全程只有"前进"或"停"，无方向反转
- 与 `car_run` 的"前进"方向一致，切换时无前后动

#### 改动 2：方向锁定

新增 `turn_dir_locked` 变量，进入转弯模式时用 `last_error` 符号锁定方向：

```c
if (last_error > 0.5f)       turn_dir_locked = true;   // 线在左 → 左转
else if (last_error < -0.5f) turn_dir_locked = false;  // 线在右 → 右转
```

- `last_error` 是丢线前最后一次有效巡线的误差，综合所有传感器，比 `left_flage`（仅基于 s1/s8）更可靠
- 转弯过程中不再读取方向，避免扫线抖动导致方向反转
- 0.5 阈值避免 error 在 0 附近抖动

#### 改动 3：退出防抖

```c
if (!s4 && !s5) {                    // s4 且 s5 都有黑线
    turn_exit_cnt++;
    if (turn_exit_cnt >= 2) {        // 连续 2 次才退出
        trun_flage = false;
        turn_exit_cnt = 0;
    }
} else {
    turn_exit_cnt = 0;
}
```

- 原条件 `if(!s4 || !s5)` 只要一个传感器扫到线就退出
- 新条件要求 s4 **且** s5 同时有线，且连续 2 次
- 确保车真的对准新线段才退出，避免 `car_run/car_trun` 频繁切换

### 9.4 涉及文件

`APPLICATION/line_follower.c`：
- 新增 `static bool turn_dir_locked` 和 `static uint8_t turn_exit_cnt`
- `line_folower` 在 `out_num >= 3` 时用 `last_error` 锁定 `turn_dir_locked`
- `car_trun` 重写为单边驱动 + 防抖退出
- `CAR_CONTROL` 简化：转弯时只传 PWM，方向由 `turn_dir_locked` 决定

### 9.5 调试建议

- 转弯太慢：`car_trun(300)` 调大到 400-500
- 转弯太快（冲过头）：调小到 200-250
- 转弯转不过去（角度不够）：增大 `out_num` 阈值（当前 3）让进入转弯更晚，或减小 PWM
- 找到线后退出不及时：减小 `turn_exit_cnt >= 2` 为 `>= 1`
- 误退出（还没转够就退出）：增大连续次数为 `>= 3`
