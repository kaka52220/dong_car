## 1. 项目概览

### 硬件配置

| 部件 | 型号/规格 | 用途 |
|------|----------|------|
| 主控 | MSPM0G3507 (TI) | 80MHz Cortex-M0+ |
| 电机驱动 | TB6612 或同类 × 2 | 4 路直流电机 |
| 编码器 | 13 线 ×2 相 | 测速，A+B 双相 26 中断/圈 |
| 灰度传感器 | 8 路数字输出 | 循迹，经 MUX 选通 |
| IMU | MPU6050 | 姿态（当前仅显示） |
| 显示 | SSD1306 OLED 128×64 | 菜单调参 |
| 通信 | 串口 | 调试打印 |

### 软件环境

- **IDE**：Code Composer Studio (CCS)
- **配置**：SysConfig 生成 `ti_msp_dl_config.c`
- **库**：u8g2（OLED）、自写 MPU6050 驱动

### 控制结构

```
传感器 → 外环(位置PID) → 目标速度 → 内环(速度PI) → PWM → 电机
              ↑                                        ↓
              └────────── 编码器反馈 ←────────────────┘
```

**串联双环**——电赛主流方案：
- 外环：循迹 PID，输出"差速"（differential）
- 内环：电机 PI，保证"实际速度=目标速度"

## 2. 硬件映射（必背）

### 电机位置（车头换向后最新）

```
        车头方向 ↓
        ┌─────────┐
   左前 │    B    │ 右前
        │         │
   左后 │    A    │ 右后
        └─────────┘
   左侧: A(后), B(前)    右侧: C(前), D(后)
```

| 电机 | 位置 | PWM 通道 | 编码器 |
|------|------|---------|--------|
| A | 左后 | PWMA (TIMG8 CCP0) | E1 (GPIOA) |
| B | 左前 | PWMB (TIMG7 CCP1) | E2 (GPIOA) |
| C | 右前 | PWMC (TIMG12 CCP1) | E3 (GPIOB) |
| D | 右后 | PWMD (TIMG6 CCP1) | E4 (GPIOB) |

### 电机方向极性（MOTOR_CONTROL 测试正确）

**每个电机的"前进"对应不同的 IN1/IN2 组合**（因左右侧对称安装）：

| 电机 | 前进时 | 后退时 |
|------|--------|--------|
| A | AIN1=1, AIN2=0 | AIN1=0, AIN2=1 |
| B | BIN2=1, BIN1=0 | BIN2=0, BIN1=1 |
| C | CIN2=1, CIN1=0 | CIN2=0, CIN1=1 |
| D | DIN1=1, DIN2=0 | DIN1=0, DIN2=1 |

> **记忆口诀**：A、D 是 IN1 高前进；B、C 是 IN2 高前进。

### 传感器布局

```
s1  s2  s3  s4  s5  s6  s7  s8
←左                              右→
权重: 7   5   3   1  -1  -3  -5  -7
```

- s1 最左，权重 +7（线在左 → error 正 → 左转）
- s8 最右，权重 -7
- s4、s5 是中心两个，用于判断"对准线"

### 传感器极性

- `raw=0`（传感器输出低）→ 检测到黑线 → `s = !raw = 1`
- 但代码里 `black_cnt` 用 `!data[i]` 计数（s=0 为黑线）
- **结论**：`s=0` 表示黑线（有线），`s=1` 表示白色（无线）

> 注意这个极性有点绕，以 `black_cnt` 的计算逻辑为准。

## 4. 主循环详解（empty.c）

```c
while (1)
{
    OLED_SHOW(&u8g2);           // 1. OLED 菜单刷新（耗时 30-50ms）
    line_follower_update();     // 2. 读 8 路灰度传感器 → s1~s8
    if(stop_flag) car_stop();   // 3. 三态机
    else CAR_CONTROL();
    // 4. 每 1 秒串口打印一次状态
    {
        if (tick_ms - last_sensor_ms >= 1000) {
            sprintf(buf, "S:%d%d%d%d%d%d%d%d L:%.0f R:%.0f mm/s Y:%.1f\r\n",
                s1, s2, s3, s4, s5, s6, s7, s8,
                calculate_motor_speed('A'),  // 左轮速度
                calculate_motor_speed('C'),  // 右轮速度
                d->yaw_deg);
        }
    }
}
```

**关键点**：
- 主循环周期 ≈ 30-50ms（OLED 刷新是瓶颈）
- 编码器采样 20ms（ISR 驱动，和主循环异步）
- 串口打印 1 秒一次（不影响控制）

## 5. 循迹模块详解（line_follower.c）

### 5.1 line_follower_update() — 读传感器

```c
void line_follower_update(void) {
    uint8_t raw[8];
    Grayscale_Sensor_Read_All(raw);   // MUX 选通 + 读 8 路
    s1 = !raw[0];  s2 = !raw[1]; ...  // 取反映射
}
```

8 路传感器通过 MUX（74HC4051 之类）选通，逐个读取。每次读有 50µs 延时（等稳定），8 路共 400µs。

### 5.2 line_folower() — 位置 PID + 丢线处理

这是**外环**，核心函数。处理流程：

```
第1步: 加权求和 → error
第2步: 全白检测 → 丢线处理
第3步: 归一化 error + 更新 left_flage
第4步: 误差死区 (|error|<1 → 0)
第5步: 积分限幅 (±100)
第6步: 位置式 PID → output
第7步: 输出限幅 (±400)
返回 output (即 differential)
```

#### 第1步：加权求和

```c
const int8_t weight[8] = {7, 5, 3, 1, -1, -3, -5, -7};
for (i = 0; i < 8; i++) {
    if (!data[i]) { sum_w += weight[i]; black_cnt++; }  // s=0 为黑线
}
error = sum_w / black_cnt;   // 归一化加权平均
```

**好处**：无论看到几个传感器，error 都反映线的中心位置。
- 线在正中（s4,s5）→ error ≈ 0
- 线偏左（s1）→ error ≈ +7
- 线偏右（s8）→ error ≈ -7

#### 第2步：丢线处理（核心！）

```c
if (black_cnt == 0) {                    // 全白，丢线
    actual_speed = base_speed / 2;       // 降速到 50%
    if (out_num == 0) {
        Velocity_ResetAll(actual_speed * 4);  // 重置 PI 积分
    }
    out_num++;
    if (out_num >= 2) {                  // 连续 2 次丢线 → 进转弯
        trun_flage = true;
        // 用 last_error 符号锁定转弯方向
        if (last_error > 1.0f)       turn_dir_locked = true;   // 左转
        else if (last_error < -1.0f) turn_dir_locked = false;  // 右转
    }
    integral = 0;
    return last_valid_diff * 0.5f;       // 保持差速方向，幅度减半
}
```

**设计要点**：
1. **降速**：`actual_speed = base_speed / 2`，避免丢线窗口直行冲
2. **续行**：`return last_valid_diff * 0.5f`，保持上次差速方向（不归零直行）
3. **2 次阈值**：`out_num >= 2`，1 次缓冲防瞬态误触发
4. **方向锁定**：用 `last_error` 符号（综合 8 路），比单传感器可靠
5. **CV 重置**：第一周期重置 PI 积分，匹配降速后的 target

#### 第6步：位置式 PID

```c
float output = kp * error + ki * integral + kd * (error - last_error);
```

- `kp`：比例，主要响应
- `ki`：积分，消除稳态误差
- `kd`：微分，抑制超调

### 5.3 car_trun() — 差速倒退转弯

```c
void car_trun(int trun_pwm) {
    if (turn_dir_locked) {   // 左转
        // 右轮 C,D 正常前进，左轮 A,B 一半速度倒退
        MOTOR_RAW(-trun_pwm/2, -trun_pwm/2, trun_pwm, trun_pwm);
    } else {                 // 右转
        // 左轮 A,B 正常前进，右轮 C,D 一半速度倒退
        MOTOR_RAW(trun_pwm, trun_pwm, -trun_pwm/2, -trun_pwm/2);
    }

    if (!s4 && !s5) {        // 中心传感器看到线 → 退出转弯
        trun_flage = false;
        Velocity_ResetAll(base_speed * 4);  // 重置 CV，防小冲
    }
}
```

**转弯半径对比**：
| 方式 | 半径 | 适用 |
|------|------|------|
| 单边驱动（一边停） | ≈ 轴距/2 = 7.5cm | 弯道 |
| **差速倒退**（一边倒退） | ≈ 2.5cm | **直角** ✓ |
| 原地旋转（全速对冲） | ≈ 0 | 原地调头（太剧烈） |

### 5.4 CAR_CONTROL() — 模式切换

```c
void CAR_CONTROL(void) {
    if (!trun_flage) {
        car_run(actual_speed, line_folower(kp, ki, kd));  // 巡线
    } else {
        car_trun(200);  // 转弯，PWM=200
    }
}
```

**注意**：用 `actual_speed` 而非 `base_speed`，这样丢线降速能生效。

## 6. 电机控制详解（electrical_machinery.c）

### 6.1 MOTOR_CONTROL() — 内环 PI 速度控制

```c
void MOTOR_CONTROL(int TargetA, int TargetB, int TargetC, int TargetD) {
    // 4 路独立 PI
    int CVA = Velocity_A(TargetA, calculate_motor_speed('A'));
    int CVB = Velocity_B(TargetB, calculate_motor_speed('B'));
    ...
    // 根据 CV 正负设方向，PWM = |CV|
    DL_TimerG_setCaptureCompareValue(PWMA_INST, CVA, ...);
}
```

### 6.2 Velocity_X() — 增量式 PI

```c
static int cv_A=0, lb_A=0;   // CV 和 Last_bias（文件作用域）

int Velocity_A(int Target, int Current) {
    int Bias = Target - Current;
    cv_A += Velcity_Kp*(Bias - lb_A) + Velcity_Ki*Bias;  // 增量式
    if (cv_A > 999)  cv_A = 999;    // 限幅
    if (cv_A < -999) cv_A = -999;
    lb_A = Bias;
    return cv_A;
}
```

**增量式 PI 公式**：
```
Δu = Kp*(e[n] - e[n-1]) + Ki*e[n]
u[n] = u[n-1] + Δu
```

**特点**：
- 不需要单独积分项，`Ki*e[n]` 本身就是积分增量
- CV 直接作为 PWM 值（正值前进，负值后退）
- 跨调用累积（static），需要 `Velocity_ResetAll` 重置

### 6.3 Velocity_ResetAll() — 重置 PI 积分

```c
void Velocity_ResetAll(int seed_target) {
    int seed_cv = seed_target * 2.2f;   // 经验值：PWM ≈ speed × 2.2
    if (seed_cv > 800)  seed_cv = 800;
    cv_A = cv_B = cv_C = cv_D = seed_cv;
    lb_A = lb_B = lb_C = lb_D = 0;
}
```

**何时调用**：
1. 丢线第一周期：`Velocity_ResetAll(actual_speed * 4)` 匹配降速
2. 退出转弯时：`Velocity_ResetAll(base_speed * 4)` 匹配正常速度

**为什么需要**：target 突变时，CV 还停留在旧值，PI 需要数十周期收敛，期间电机过度驱动（飙速/小冲）。

### 6.4 MOTOR_RAW() — 绕过 PI 直接控制

```c
void MOTOR_RAW(int A, int B, int C, int D) {
    // 正值→前进方向，负值→后退方向，0→停
    // PWM = |值|，限幅 999
}
```

**用于转弯**（`car_trun`），因为转弯时不需要精确速度控制，直接给 PWM 更简单。

**极性同步**：MOTOR_RAW 的每个电机 IN1/IN2 参数对调情况和 MOTOR_CONTROL 一致：
- A、D：对调（IN1 高前进）
- B、C：不对调（IN2 高前进）

### 6.5 car_run() — 速度映射

```c
void car_run(int base_speed_pct, float differential) {
    int base_speed = base_speed_pct * 4;           // 50% → 200 mm/s
    float left  = base_speed - differential / 2.0f;
    float right = base_speed + differential / 2.0f;
    // 限幅 ±400
    MOTOR_CONTROL(left, left, right, right);       // A,B=左轮, C,D=右轮
}
```

**差速原理**：
- `differential > 0`（线在左）→ 左轮慢、右轮快 → 左转
- `differential < 0`（线在右）→ 左轮快、右轮慢 → 右转

## 7. 编码器与速度计算（interrupt.c）

### 编码器中断

```c
// GPIO 中断：A/B 相触发，判方向，temp_count++
void GROUP1_IRQHandler(void) {
    case GPIO_ENCODER_E1A_IIDX:
        if (!DL_GPIO_readPins(GPIOA, GPIO_ENCODER_E1B_PIN))
            motor_encoderA.temp_count--;
        else
            motor_encoderA.temp_count++;
        break;
}

// 20ms 定时器中断：temp_count → count，清零 temp_count
void TIMER_TICK_INST_IRQHandler(void) {
    motor_encoderA.count = motor_encoderA.temp_count;
    motor_encoderA.temp_count = 0;
}
```

### 速度计算

```c
#define PULSE_TO_MMS 14.03f   // π×65mm / (13×2×28) × 50Hz

float calculate_motor_speed(char motor) {
    int pulse = get_encoder_count(motor);
    int mag = (pulse >= 0) ? pulse : -pulse;    // 取绝对值
    float instant = mag * PULSE_TO_MMS;
    speed = 0.5 * instant + 0.5 * speed;        // 一阶低通滤波
    return speed;
}
```

**参数推导**：
- 编码器 13 线，A+B 双相 = 26 中断/圈
- 减速比 1:28 → 车轮每圈 = 26×28 = 728 中断
- 1 中断位移 = π×65/728 = 0.28mm
- 采样 50Hz → 速度 = 脉冲数 × 0.28 × 50 = 脉冲数 × 14.03 mm/s

## 8. 完整控制逻辑流

### 场景1：正常巡线

```
传感器看到线 → line_folower 算 error → PID 输出 differential
→ car_run(base, diff) → 左右目标速度 → MOTOR_CONTROL(PI) → PWM → 电机
→ 编码器反馈速度 → PI 调整 PWM → 速度稳定
```

### 场景2：直角转弯（丢线）

```
1. 车到直角顶点，前方无线 → black_cnt=0（丢线）
2. 第1周期丢线：
   - actual_speed = base_speed/2（降速）
   - Velocity_ResetAll 匹配降速
   - return last_valid_diff*0.5（保持差速，不直冲）
   - car_run(降速, 差速减半) → 减速差速转弯
3. 第2周期丢线（out_num>=2）：
   - trun_flage = true（进转弯模式）
   - turn_dir_locked 用 last_error 符号锁定
4. 转弯模式：
   - car_trun(200) → MOTOR_RAW 差速倒退
   - 一边前进 PWM=200，另一边倒退 PWM=100
   - 转弯半径 ≈ 2.5cm
5. 中心传感器 s4,s5 看到线 → 退出转弯：
   - trun_flage = false
   - Velocity_ResetAll(base_speed*4) 重置 CV
6. 回到正常巡线
```

### 场景3：停车

```
stop_flag=true → car_stop() → car_run(0, 0) → target=0 → PI 减速到 0
```

## 9. 关键参数表

### 循迹 PID 参数（draw.c，OLED 可调）

| 参数 | 当前值 | 作用 | 调参建议 |
|------|--------|------|---------|
| kp | 7.0 | 比例（主要响应） | 弯道跟不上→增大；直线抖动→减小 |
| ki | 0.2 | 积分（消静差） | 直线偏一侧→增大；震荡→减小 |
| kd | 0.5 | 微分（抑超调） | 弯道冲过头→增大 |
| pid_rate | 0.1 | 调参步长 | — |
| OMAX | 400 | 差速限幅 | 转向不够→增大 |

### 速度 PI 参数（electrical_machinery.c）

| 参数 | 当前值 | 作用 |
|------|--------|------|
| Velcity_Kp | 0.5 | 速度环比例 |
| Velcity_Ki | 0.3 | 速度环积分 |
| PWM_MAX | 999 | PWM 满幅 |

### 速度参数

| 参数 | 当前值 | 作用 |
|------|--------|------|
| base_speed | 50 (%) | 基础速度（50% → 200mm/s）|
| actual_speed | 动态 | 丢线时降到 base_speed/2 |

## 10. 演进历史

### 第一阶段：丢线飙速修复（7-19）

**问题**：丢线后速度从 200mm/s 飙升到 500-900mm/s。

**根因**：
1. `car_trun(100)` 里 `×100` 放大导致 PWM 钳到 999（全速）
2. `Velocity_X` 的 static CV 在 target 突变时积分饱和

**修复**：
- 去掉 `×100`，`car_trun(300)` → `car_trun(200)`
- 新增 `Velocity_ResetAll` 重置 PI 积分

### 第二阶段：直角循迹转弯优化（7-20）

**问题**：原地旋转转弯导致"前后动"。

**修复**：
- 改单边驱动（一边停一边转）
- 方向锁定（`turn_dir_locked`，用 `last_error` 符号）
- 退出防抖（`s4 && s5`）

### 第三阶段：车头换向（7-20）

**变更**：物理车头转 180°，电机位置全变。

**修复**：
- 用户改 `MOTOR_CONTROL` 极性适配新车头
- `MOTOR_RAW` 同步极性（A,D 对调；B,C 不对调）
- `car_run` 左右映射互换
- `empty.c` 打印 L/R 互换

### 第四阶段：小冲修复（7-20）

**问题**：丢线时往前冲一段、找到线时也冲一段。

**根因**：
1. 丢线窗口 `return 0` → 等速直行
2. 退出转弯时 CV 残留旧值

**修复**：
- 丢线续行：`return last_valid_diff*0.5`（保持差速）
- 丢线降速：`actual_speed = base_speed/2`
- `out_num >= 3` → `>= 2`（减少窗口）
- 退出转弯调 `Velocity_ResetAll`

### 第五阶段：差速倒退转弯（7-20）

**问题**：单边驱动转弯半径太大，直角出线。

**修复**：
- 一边前进 + 另一边一半速度倒退
- 转弯半径从 7.5cm 降到 2.5cm

## 11. 电赛优化建议（冲刺省一）

### 优先级 1：分段速度（投入小，收益大）

**问题**：当前固定 `base_speed=50%`，直道不够快、弯道太快冲出。

**方案**：根据 error 大小动态调速：
```c
// 在 line_folower 里
if (abs(error) < 2)      actual_speed = base_speed;       // 直道全速
else if (abs(error) < 5) actual_speed = base_speed * 0.7; // 弯道降速
else                     actual_speed = base_speed * 0.5; // 急弯再降
```

**效果**：直道快、弯道稳，圈速提升 20-30%。

### 优先级 2：MPU6050 接入控制环

**现状**：IMU 只用于 OLED 显示，没参与控制。

**方案**：用 yaw 角走绝对直线 + 精确转 90°：
```c
// 巡线时叠加 yaw 反馈，走更直
float yaw_error = target_yaw - current_yaw;
differential += Kp_yaw * yaw_error;

// 直角转弯时用 yaw 精确控制角度
if (turn_dir_locked) {
    target_yaw += 90;  // 左转 +90°
    while (abs(yaw_error) > 2) {
        car_trun(200);  // 持续转弯直到转够 90°
    }
}
```

**效果**：直线不跑偏，直角精确 90°，不依赖传感器扫线。

### 优先级 3：入弯前预减速

**方案**：用编码器里程检测"即将到弯道"，提前减速：
```c
// 检测到 error 开始变大（入弯前兆）
if (abs(error - last_error) > threshold) {
    actual_speed = base_speed * 0.6;  // 预减速
}
```

或者用超声波/红外检测前方弯道。

### 优先级 4：丢线容错（假丢线处理）

**现状**：丢线 2 次就进转弯，传感器瞬态误读会误触发。

**方案**：丢线后先减速续行 0.3 秒，找不到再进转弯：
```c
if (black_cnt == 0) {
    if (out_num < 5) {  // 前 5 周期（~250ms）续行
        return last_valid_diff * 0.5f;
    } else {            // 超过 5 次才确认真丢线
        trun_flage = true;
    }
}
```

### 优先级 5：接入 velocity_pid.c 新模块

**现状**：`APPLICATION/velocity_pid.c` 是更完善的 PI（含抗饱和、死区、堵转保护），但未接入。

**方案**：在 20ms ISR 里调用 `VelPID_UpdateAll`，主循环只设 target：
```c
void TIMER_TICK_INST_IRQHandler(void) {
    // 原有编码器采样...
    VelPID_UpdateAll(get_encoder_count('A'), ...);
}
// 主循环只设目标
VelPID_SetAllTargets(left, left, right, right);
```

**效果**：PI 在 ISR 里跑，周期固定 20ms，不受主循环 OLED 延迟影响。

### 优先级 6：里程计

**方案**：编码器累计脉冲算走过的距离，用于定点停车、计圈：
```c
float total_distance = 0;
// 在 20ms ISR 里
total_distance += instant_speed * 0.02;  // 累计位移
```

## 14. 关键约定与避坑

### 电机极性（最重要！）

```
A、D：IN1 高 = 前进
B、C：IN2 高 = 前进（极性相反）
```

**避坑**：换车头、换电机接线后，必须同步更新：
1. `MOTOR_CONTROL` 的 IN1/IN2 极性
2. `MOTOR_RAW` 的 IN1/IN2 参数对调情况
3. `car_run` 的左右映射
4. `empty.c` 的 L/R 打印

### MOTOR_RAW 参数顺序

```c
MOTOR_RAW(A, B, C, D)
//     左后, 左前, 右前, 右后
```

**注意**：不是"左前左后右前右后"，而是按电机编号 A-B-C-D。

### 传感器极性

```
s=0 → 黑线（有线）
s=1 → 白色（无线）
```

代码里用 `!data[i]` 判断黑线，`black_cnt` 计数黑线数量。

### static 变量跨调用累积

`Velocity_X` 的 CV（`cv_A` 等）是文件作用域 static，跨调用累积。target 突变时必须调 `Velocity_ResetAll` 重置，否则积分饱和导致飙速/小冲。

### 转弯绕过 PI

`car_trun` 用 `MOTOR_RAW`，不走 PI。退出转弯时 CV 停在旧值，必须调 `Velocity_ResetAll` 重置，否则回到 `car_run` 时小冲。

### OLED 是性能瓶颈

`OLED_SHOW` 耗时 30-50ms，主循环周期受此限制。如果控制不稳，考虑：
1. 减少 OLED 刷新频率（每 5 个主循环刷一次）
2. 把 PI 移到 20ms ISR（见优先级 5）
