# 对话总结：丢线与找线后"小冲"问题专题

> **聚焦**：为什么丢线时会往前冲一段？为什么找到线后也会冲一段？
>
> **背景**：7-19 到 7-20 期间，围绕四驱循迹小车的调试对话。
>
> **目的**：把"小冲"这个问题的根因、演进、最终解决方案讲透，方便你以后自己排查类似问题。

---

## 一、问题全貌

整个调试过程中，"小冲"问题经历了几个阶段，每次表现不同、根因不同：

| 阶段 | 现象 | 根因 | 严重程度 |
|------|------|------|---------|
| 最初 | 丢线后飙速到 900mm/s | PWM ×100 放大 + PI 积分饱和 | 严重 |
| 中期 | 丢线后往前冲一段再转弯 | 丢线窗口 `return 0` → 等速直行 | 中等 |
| 中期 | 找到线后冲一段再巡线 | 退出转弯时 CV 残留旧值 | 中等 |
| 最终 | 平滑过渡 | 多重防护机制 | 已解决 |

下面按"丢线小冲"和"找线小冲"两条线分别讲。

---

## 二、丢线时"往前冲一段"的原因与解决

### 2.1 现象描述

> **用户原话**："丢线的时候小车会往前面小冲一段然后再转弯"

车到直角顶点，前方无线（丢线），本应立即转弯，但车却**先直行冲一段**（约 100ms），然后才开始转弯。

### 2.2 根因分析

问题出在 `line_folower()` 的丢线处理逻辑。原代码：

```c
if (black_cnt == 0) {           // 丢线
    if (out_num == 0) {
        Velocity_ResetAll(base_speed * 4);   // 重置 CV
    }
    out_num++;
    actual_speed = 0;
    if (out_num >= 3) {          // 连续 3 次才进转弯
        trun_flage = true;
    }
    integral = 0;
    return 0;                    // ← 关键：differential = 0
}
```

**问题链**：

1. **直行窗口**：`out_num >= 3` 意味着丢线后前 2 个周期（约 100ms）不进转弯，仍走 `car_run`。
2. **`return 0`**：differential=0 → `car_run(base_speed, 0)` → 四轮等速 target=200mm/s → **直行**。
3. **CV 重置到 200**：`Velocity_ResetAll(base_speed * 4)` 把 CV 重置到 440（对应 200mm/s）。
4. **电机从差速突变等速**：丢线前车在差速转弯（左右轮速度不同），丢线瞬间突然变成四轮等速 200 → **直冲**。

简单说：**丢线后本该减速转弯，代码却让车等速直行了 2 个周期**。

### 2.3 解决方案

三个改动配合：

#### 改动 1：丢线续行——保持差速，不归零

```c
// 原来：
return 0;

// 改为：
return last_valid_diff * 0.5f;   // 保持上次差速方向，幅度减半
```

新增 `static float last_valid_diff = 0;`，在正常巡线返回前保存：
```c
last_valid_diff = output;
return output;
```

**效果**：丢线后车继续差速转弯（减速），而不是直行。

#### 改动 2：丢线降速

```c
// 原来：
actual_speed = 0;           // 实际没人用，死代码

// 改为：
actual_speed = base_speed / 2;   // 降速到 50%
```

同时 `CAR_CONTROL` 改用 `actual_speed`：
```c
// 原来：
car_run(base_speed, line_folower(kp, ki, kd));

// 改为：
car_run(actual_speed, line_folower(kp, ki, kd));   // 丢线降速生效
```

**效果**：丢线后车速降一半，即使有少量直行趋势也不会冲太远。

#### 改动 3：减少直行窗口

```c
// 原来：
if (out_num >= 3) {     // 2 个周期直行窗口

// 改为：
if (out_num >= 2) {     // 1 个周期直行窗口
```

**效果**：直行窗口从 ~100ms 减到 ~50ms，几乎无感。

### 2.4 修复后的丢线流程

```
丢线第1周期：
  actual_speed = base_speed/2（降速）
  Velocity_ResetAll(actual_speed*4)（CV 重置到匹配降速）
  return last_valid_diff*0.5（保持差速转弯）
  → car_run(降速, 差速减半) → 减速差速转弯（不直冲）

丢线第2周期（out_num>=2）：
  trun_flage = true → 进转弯模式
  → car_trun(200) 差速倒退转弯
```

## 三、找到线后"冲一段"的原因与解决

### 3.1 现象描述

> **用户原话**："找到线后也会小冲一段然后再巡线"

车在转弯模式（单边驱动/差速倒退），中心传感器看到线后退出转弯，回到正常巡线。但**回巡线的瞬间车会冲一下**，然后才稳定。

### 3.2 根因分析

问题出在 `car_trun` 退出转弯时的 CV 状态。

**关键背景**：
- 转弯用 `MOTOR_RAW`，**绕过 PI**（不走 `Velocity_X`）
- `Velocity_X` 的 CV 是 static 变量，**跨调用累积**
- 转弯期间 PI 没被调用，CV **停留在旧值**

**问题链**：

1. 进转弯前，`Velocity_ResetAll` 把 CV 重置到 440（对应 200mm/s）
2. 转弯期间，`MOTOR_RAW` 直接设 PWM，CV 不更新，停在 440
3. 转弯时一侧轮子停（PWM=0），另一侧前进/倒退
4. 退出转弯，回到 `car_run`，target=200（正常巡线）
5. **但此时**：
   - 停着的那侧轮子 current=0（编码器没脉冲）
   - CV=440（旧值）
   - `Velocity_X(200, 0)`：Bias = 200-0 = 200
   - ΔCV = Kp*(200-0) + Ki*200 = 100 + 60 = 160
   - CV = 440 + 160 = 600 → **PWM=600（60% 占空比）**
6. 电机突然以 60% 占空比运转 → **小冲一段**
7. PI 需要几个周期才把 CV 收敛回匹配 200mm/s 的值（~440）

简单说：**转弯时 PI 被绕过，CV 停在旧值，退出转弯时 CV 和新 target 不匹配，PI 短暂过度驱动电机**。

### 3.3 解决方案

在 `car_trun` 退出转弯时，调 `Velocity_ResetAll` 重置 CV：

```c
void car_trun(int trun_pwm) {
    // ... 转弯逻辑 ...

    if (!s4 && !s5) {                    // 中心传感器看到线
        trun_flage = false;              // 退出转弯
        Velocity_ResetAll(base_speed * 4);   // ← 关键：重置 CV
    }
}
```

**效果**：CV 被重置到 440（匹配 base_speed=200mm/s），PI 从正确初值开始，不会过度驱动。

### 3.4 修复前后对比

| 时刻 | 修复前 CV | 修复前 PWM | 修复后 CV | 修复后 PWM |
|------|----------|-----------|----------|-----------|
| 转弯中 | 440（停滞） | 0（MOTOR_RAW 控制）| 440（停滞）| 0 |
| 退出瞬间 | 440 | **600（冲！）** | **440（重置）** | **440（平稳）** |
| 1 周期后 | ~550（收敛中）| ~550 | ~440（稳定）| ~440 |

## 四、两个"小冲"的本质区别

虽然都叫"小冲"，但根因完全不同：

| 对比项 | 丢线小冲 | 找线小冲 |
|--------|---------|---------|
| **发生时机** | 丢线瞬间 | 退出转弯瞬间 |
| **根因** | `return 0` 导致等速直行 | CV 残留旧值，PI 过度驱动 |
| **涉及模块** | `line_folower`（外环） | `Velocity_X`（内环） |
| **修复方式** | 续行+降速+减窗口 | 退出时重置 CV |
| **修复函数** | `line_folower` | `car_trun` |

**共同点**：都是"target 突变时，控制器的状态没跟上"。
- 丢线小冲：target 从差速突变到等速，differential 没跟上
- 找线小冲：target 从转弯突变到巡线，CV 没跟上

## 五、背后的控制理论

这两个问题本质上是**控制系统状态不连续**导致的冲击。

### 5.1 增量式 PI 的状态依赖

增量式 PI 的公式：
```
u[n] = u[n-1] + Kp*(e[n]-e[n-1]) + Ki*e[n]
```

输出 `u[n]` 依赖上一时刻 `u[n-1]`。如果 `u[n-1]` 是个错误的值（比如转弯时 CV 停滞），新输出也会跟着错。

### 5.2 模式切换的状态跳变

系统有三个模式：停车、巡线、转弯。模式切换时：
- 巡线 → 转弯：`car_run` → `car_trun`，PI 被绕过
- 转弯 → 巡线：`car_trun` → `car_run`，PI 重新启用，但 CV 是旧值

**状态跳变**导致控制不连续 → 冲击。

### 5.3 解决思路：状态重置

在模式切换的瞬间，**显式重置控制器状态**，让新模式从合理的初值开始：

```c
// 丢线时（进转弯前）：
Velocity_ResetAll(actual_speed * 4);   // 重置到匹配降速

// 退出转弯时（回巡线）：
Velocity_ResetAll(base_speed * 4);     // 重置到匹配正常速度
```

这是控制工程里的**bumpless transfer（无冲击切换）**思想。

## 六、完整防护机制

最终代码里，"小冲"防护有四道防线：

```
丢线瞬间 ──→ 第1道：降速（actual_speed = base_speed/2）
            第2道：续行（return last_valid_diff*0.5，不直行）
            第3道：CV 重置（Velocity_ResetAll 匹配降速）

退出转弯 ──→ 第4道：CV 重置（Velocity_ResetAll 匹配正常速度）
```

每一道防线解决一个具体问题：

| 防线 | 解决的问题 |
|------|-----------|
| 降速 | 丢线窗口即使有直行趋势，速度低也不会冲太远 |
| 续行 | 不归零差速，车继续转弯而非直行 |
| 丢线时重置 CV | CV 匹配降速后的 target，PI 不过度驱动 |
| 退出时重置 CV | CV 匹配正常 target，PI 不过度驱动 |

- 先修丢线小冲（改 `line_folower`）
- 再修找线小冲（改 `car_trun`）
- 验证时分别观察"丢线瞬间"和"找线瞬间"

## 九、经验总结

### 9.1 模式切换是 bug 高发区

任何时候系统有多个模式（巡线/转弯/停车），模式切换的瞬间都要检查：
- 控制器的状态（CV、integral 等）是否连续？
- target 是否突变？
- 有没有需要重置的累积量？

### 9.2 static 变量是双刃剑

`Velocity_X` 的 static CV 跨调用累积，好处是增量式 PI 不需要单独积分项，坏处是 target 突变时容易饱和。**用 static 累积的地方，一定要提供重置接口**。

### 9.3 "return 0" 不是万能的

丢线时 `return 0` 看起来合理（差速归零，等速直行），但实际上导致车直冲。**归零不一定是安全的，保持上次状态（续行）往往更平稳**。

### 9.4 控制器的"状态"要和管理

不只是输入输出，控制器的**内部状态**（CV、integral、last_error）也要在模式切换时管理好。这是从"能跑"到"跑得稳"的关键。

**最后**：这两个"小冲"问题本质都是"控制系统状态不连续"。理解了这一点，以后遇到类似的"切换时冲击"问题，第一反应就是检查控制器状态（CV、积分项等）是否需要重置。这是控制工程里很通用的思想，不止电赛用得上。
