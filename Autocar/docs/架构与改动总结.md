# 项目架构与改动总结

> MCU: MSPM0G3507 | 传感器: PCF8575DBR + MPU6050 | 外设: OLED(SSD1306) + 4路电机

---

## 一、最终架构图

```
┌──────────────────────────────────────────────────┐
│                  SysTick 1ms 中断                  │
│  tick_ms++                                        │
│  每 5 次 → mpu6050_update()   ← dt 精确 5ms       │
│  (I2C1 → MPU6050 读原始数据 + Mahony 姿态融合)     │
└──────────────────────────────────────────────────┘
                        ↓ 写入全局 g_data
┌──────────────────────────────────────────────────┐
│                   主循环 while(1)                  │
│                                                    │
│  OLED_SHOW()          ← u8g2 全屏渲染，不阻塞 IMU  │
│  line_follower_update() ← I2C0 → PCF8575 读 8 路   │
│  CAR_CONTROL()        ← PID 巡线 + 电机控制        │
│  mpu6050_get_data()   ← 取角度（每 1s 串口打印）    │
└──────────────────────────────────────────────────┘
```

**核心原则：IMU 采样由硬件定时器驱动，主循环怎么阻塞都不影响角度精度。**

## 三、关键原理

### 3.1 PCF8575（循迹传感器）I2C 通信

PCF8575 是 **16 位 IO 扩展芯片，没有寄存器概念**。

```
错误做法（mspm0_motor_i2c_read）:
  START → 0x40(W) → 0x41(被当成输出数据) → RESTART → 读 → 超时

正确做法（mspm0_motor_i2c_read_direct）:
  START → 0x41(R) → data0 → data1 → STOP
```

**MSPM0 SDK 地址处理规则**：传入 7 位纯地址 `0x20`，SDK 内部 `(addr << 1) | direction` 自动生成总线字节 `0x41`。

**硬件要求**：PCF8575 的 A0(Pin1)、A1(Pin2)、A2(Pin3) 必须全部接地，地址才是 `0x20`。悬空会导致地址漂移。

### 3.2 MPU6050 角度精度

`mpu6050_update()` 内部 dt 限幅：
```c
dt = (tick_ms - last_update) * 0.001;
if (dt < 4ms || dt > 50ms) dt = 0.01;  // 超限强制 10ms → 丢角度！
```

**问题**：主循环中 OLED 全屏渲染耗时 30-50ms，如果 IMU 采样也在主循环里，dt 极易超过 50ms 被限幅。

**解决**：SysTick 中断硬件定时采样，dt 永远 = 5ms，主循环无论如何阻塞都不影响。

### 3.3 返回值速查

| 函数 | 返回值 | 含义 |
|------|--------|------|
| `mspm0_motor_i2c_read_direct` | 0 | 成功 |
| | -1 | 超时（总线卡死） |
| | -2 | NACK（从机不应答地址） |
| | -3 | IDLE 超时（控制器忙） |

## 四、串口输出格式

每 1 秒一行：
```
S:00111100 Y:45.3 P:-2.1 R:1.8
│  │       │       │       │
│  8 路传感器 偏航角  俯仰角  翻滚角
│  (0=黑 1=白)
```
