# 转弯方向判断方案对比

> 丢线后判断左转还是右转的三种方案，出问题时可按此文档切换。

---

## 方案对比

### 方案 A：last_error 符号（已弃用）

```c
if (last_error > 1.0f)       turn_dir_locked = true;   // 左转
else if (last_error < -1.0f) turn_dir_locked = false;  // 右转
```

- **原理**：用丢线前最后一次有效巡线的 error 符号
- **error 计算**：所有 8 个传感器加权平均，权重 s1=+7...s8=-7
- **问题**：
  - 锐角弯时 s3/s4 检测到线，权重 +3/+1 让 error 偏正 → 误判左转
  - 直角弯丢线前 error≈0 → |error|<=1.0 不更新 → 保持上次方向（可能错）
  - 连锁错误：上次方向错，后续都跟着错
- **适用**：线从某一侧明确消失的场景

### 方案 B：s1s2 vs s7s8 对比（当前使用）

```c
// 第3步（正常巡线时持续更新）
int left_cnt  = (!s1) + (!s2);   // 左侧有线数量
int right_cnt = (!s7) + (!s8);   // 右侧有线数量
if (left_cnt > right_cnt)       left_flage = true;
else if (right_cnt > left_cnt)  left_flage = false;

// 丢线时
turn_dir_locked = left_flage;
```

- **原理**：只看最外侧 4 个传感器，两侧各 2 个统计数量对比
- **优点**：
  - 不受中间 s3/s4/s5/s6 干扰 → 解决锐角弯转反
  - 两个传感器比一个稳定 → 降低噪声影响
  - 正常巡线时持续更新 → 有多次积累，不只依赖丢线瞬间
- **问题**：线在正前方消失时（s1s2 和 s7s8 都没线），保持上次方向
- **适用**：直角弯、锐角弯，线从某一侧消失的场景

### 方案 C：IMU yaw 精确转弯（备用，最可靠）

```c
// 丢线时记录目标角度
float target_yaw = mpu6050_get_yaw();
if (left_flage) target_yaw += 90.0f;   // 左转 +90°
else            target_yaw -= 90.0f;   // 右转 -90°

// 转弯时用 yaw 误差驱动
float yaw_error = target_yaw - mpu6050_get_yaw();
if (yaw_error > 2.0f) {
    car_trun(200);   // 继续转
} else {
    trun_flage = false;   // 转够了，退出
}
```

- **原理**：用 IMU yaw 精确控制转弯角度，不依赖传感器判断方向
- **优点**：
  - 方向绝对可靠（yaw 不会"转反"）
  - 转弯角度精确（90°±5°）
  - 不受传感器噪声/干扰影响
- **问题**：
  - 需要 IMU yaw 足够准（当前 5° 误差，多次转弯会累积）
  - 改动较大（car_trun 逻辑要重写）
  - 锐角弯不是 90°，需要预设角度
- **适用**：IMU 精度够高、弯道角度已知的场景

## 如何切换方案

改 `line_follower.c` 两处：

### 切到方案 A（last_error）：
```c
// 第3步保持原样（不改动 left_flage 更新逻辑）

// 丢线时（out_num >= 2 分支）改成：
if (last_error > 1.0f)       turn_dir_locked = true;
else if (last_error < -1.0f) turn_dir_locked = false;
```

### 切到方案 B（s1s2 vs s7s8，当前）：
```c
// 第3步：
int left_cnt  = (!s1) + (!s2);
int right_cnt = (!s7) + (!s8);
if (left_cnt > right_cnt)       left_flage = true;
else if (right_cnt > left_cnt)  left_flage = false;

// 丢线时：

```

## 调试建议

如果方向判断还是有问题，观察串口打印的 `S:xxxxxxxx`：
- `0` = 黑线（有线），`1` = 白色（无线）
- 丢线前最后几帧的传感器状态决定了方向判断
- 重点看 s1,s2 和 s7,s8 的值，对比是否和实际转弯方向一致

| 丢线前传感器 | 方案A判断 | 方案B判断 | 实际方向 |
|-------------|----------|----------|---------|
| S:00111100 (s3s4有线) | error=+2→左转❌ | left=0,right=0→保持 | 看弯道方向 |
| S:00111110 (s3s4有线) | error=+2→左转❌ | left=0,right=0→保持 | 看弯道方向 |
| S:00000010 (s1-s4有线) | error偏正→左转✓ | left=2,right=0→左转✓ | 左转 |
| S:01111100 (s7s8有线) | error偏负→右转✓ | left=0,right=2→右转✓ | 右转 |
| S:01111110 (正前方消失) | error≈0→保持 | left=0,right=0→保持 | 看上次 |
