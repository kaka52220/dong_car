# line_follower 循迹代码分析文档

## 1. 整体架构

```
┌─────────────────────────────────────────────────────────────┐
│                      CAR_CONTROL()                          │
│  (主控制器，每 20ms 调用一次)                                │
└──────────────────────────┬──────────────────────────────────┘
                           │
            ┌──────────────┼──────────────┐
            ▼              ▼              ▼
   line_follower_update  line_folower()  car_trun()
    (读传感器)           (PID 算差速)    (脱线转弯)
            │              │
            ▼              ▼
   mspm0_motor_i2c_read   car_run(actual_speed, output)
    (I2C 读取)                 │
                               ▼
                         MOTOR_CONTROL()
                         (发给电机 PWM)
```

## 2. 硬件连接

### 2.1 红外循迹传感器模块

通过 I2C 总线连接，从机地址 `0x20`。

| 参数 | 值 |
|---|---|
| I2C 从机地址 | `0x20` |
| 写寄存器地址 | `0x40` |
| 读寄存器地址 | `0x41` |
| 单次读取长度 | 2 字节（实际只用 Byte0） |

**引脚排列**（车头方向为前进方向）：

```
                 车头方向（前进）
           ────────────→
      s1  s2  s3  s4  s5  s6  s7  s8
       │   │   │   │   │   │   │   │
      [ ] [ ] [ ] [ ] [ ] [ ] [ ] [ ]   ← 8 路红外传感器
       └───┴───┴───┴─┬─┴───┴───┴───┘
                      │
                  ┌───┴───┐
                  │  黑线  │
                  └───────┘
```

## 4. I2C 读取：`mspm0_motor_i2c_read` 详解

### 4.1 函数原型

```c
int mspm0_motor_i2c_read(
    unsigned char slave_addr,   // 从机地址：0x20
    unsigned char reg_addr,     // 寄存器地址：0x41
    unsigned char length,       // 读取长度：2
    unsigned char *data         // 数据缓冲区：IRbuf
);
```

### 4.2 I2C 读取时序

```
主机（MSPM0）                     从机（传感器模块）
    │                                   │
    ├─ START ──────────────────────────→│
    ├─ 发送 0x20（写位）──────────────→│
    │                              ←───┤ ACK
    ├─ 发送 reg_addr (0x41) ────────→│
    │                              ←───┤ ACK
    ├─ REPEATED START ────────────────→│
    ├─ 发送 0x21（读位）──────────────→│
    │                              ←───┤ ACK
    │                              ←───┤ 发送 Byte0 (8路状态)
    ├─ ACK ──────────────────────────→│
    │                              ←───┤ 发送 Byte1 (未使用)
    ├─ NACK ─────────────────────────→│
    ├─ STOP ──────────────────────────→│
    │                                   │
```

### 4.3 读取 2 字节的原因

- **硬件强制要求**：该传感器模块的 I2C 从机固件可能将数据寄存器定义为 16 位，只读 1 字节会导致从机不释放 SDA 线
- **兼容性**：读 2 字节仅用 Byte0，对功能无影响

### 4.4 数据解析

```c
IRbuf[0] = 0b_1110_0111;   // 举例
           ││││ ││││
           ││││ │││└── s8 = 1 (最右为白)
           ││││ ││└─── s7 = 1
           ││││ │└──── s6 = 1
           ││││ └───── s5 = 0 (黑线)
           │││└─────── s4 = 0 (黑线)
           ││└──────── s3 = 1
           │└───────── s2 = 1
           └────────── s1 = 1
```

s1~s8 提取代码：
```c
s1 = (IRbuf[0] >> 7) & 0x01;
s2 = (IRbuf[0] >> 6) & 0x01;
// ...
s8 = (IRbuf[0] >> 0) & 0x01;
```

## 5. 传感器模式匹配（误差计算）

### 5.1 原理

8 个传感器排列在一条线上，检测黑色轨迹线。因为黑线宽度通常覆盖 2~3 个传感器，所以根据哪几个传感器检测到黑线，可以计算出车体相对于黑线的偏移量。

**规则**：
- 黑线在白地上 **反光弱 → 传感器输出 0**
- 白地 **反光强 → 传感器输出 1**

### 5.2 完整模式表

符号约定：**正 = 车偏右（需左转修正），负 = 车偏左（需右转修正）**

| 传感器模式 (s1~s8) | error | 图示 |
|---|---|---|
| `11100111` | **0** | `[ ][ ][ ][██][██][ ][ ][ ]` 正中 |
| `11101111` | 1 | `[ ][ ][ ][██][ ][ ][ ][ ]` 微偏右 |
| `11001111` | 2 | `[ ][ ][██][██][ ][ ][ ][ ]` 偏右 |
| `11011111` | 3 | `[ ][ ][██][ ][ ][ ][ ][ ]` 偏右较多 |
| `10011111` | 4 | `[ ][██][██][ ][ ][ ][ ][ ]` 偏右很多 |
| `10111111` | 5 | `[ ][██][ ][ ][ ][ ][ ][ ]` 严重偏右 |
| `00111111` | 6 | `[██][██][ ][ ][ ][ ][ ][ ]` 即将脱线(右) |
| `01111111` | 7 | `[██][ ][ ][ ][ ][ ][ ][ ]` 边缘脱线(右) |
| `11110111` | -1 | `[ ][ ][ ][ ][██][ ][ ][ ]` 微偏左 |
| `11110011` | -2 | `[ ][ ][ ][ ][██][██][ ][ ]` 偏左 |
| `11111011` | -3 | `[ ][ ][ ][ ][ ][██][ ][ ]` 偏左较多 |
| `11111001` | -4 | `[ ][ ][ ][ ][ ][██][██][ ]` 偏左很多 |
| `11111101` | -5 | `[ ][ ][ ][ ][ ][ ][██][ ]` 严重偏左 |
| `11111100` | -6 | `[ ][ ][ ][ ][ ][ ][██][██]` 即将脱线(左) |
| `11111110` | -7 | `[ ][ ][ ][ ][ ][ ][ ][██]` 边缘脱线(左) |
| `00000000` | 0 (特殊) | 全黑（交叉口/全线），计数等待 |

### 5.3 模式匹配的缺点

- **只覆盖了 16 种模式**，实际 8 位传感器有 256 种可能组合
- 部分重叠或过渡状态（如 `10011011`）没有任何匹配，此时 error 保持上一次的值或 0
- 更好的做法是**加权平均法**（见附录 A）

## 6. 位置式 PID 原理解析

### 6.1 什么是 PID

PID 控制器通过计算**目标值**与**实际值**之间的误差，结合三种控制作用来驱动系统：

```
                ┌─────────────────┐
                │    目标值 = 0    │ （循迹时目标始终是 0，即车正对黑线）
                └────────┬────────┘
                         │
     ┌───────────────────┼───────────────────┐
     ▼                   ▼                   ▼
┌─────────┐       ┌───────────┐       ┌──────────┐
│ P 比例项 │       │  I 积分项  │       │ D 微分项 │
│ Kp × e  │       │ Ki × ∫e   │       │ Kd × Δe │
└────┬────┘       └─────┬─────┘       └────┬─────┘
     └────────────┬─────┴─────────┬─────────┘
                  ▼               ▼
            ┌──────────┐   ┌──────────────┐
            │  求和(+) │   │ car_run 差速 │
            └──────────┘   └──────────────┘
```

### 6.2 位置式 PID 公式

```c
// 关键代码（line_follower.c:136-138）
integral += error;                                   // 积分：累计误差
float output = kp * error                            // 比例项
              + ki * integral                         // 积分项
              + kd * (error - last_error);            // 微分项
last_error = error;                                   // 保存当前误差供下次使用
return output;
```

数学表达：

```
u(t) = Kp × e(t) + Ki × Σe(k) + Kd × [e(t) - e(t-1)]
         ↑             ↑               ↑
       比例项         积分项           微分项
```

### 6.3 三项的作用

#### P（比例项）：`kp × error`

- **作用**：根据当前偏差大小，成比例地打方向盘
- **效果**：偏得越远，转向越猛
- **问题**：单独用 P 会有**稳态误差**（静差），即车永远差一点点无法完全归中

```
车向右偏 3 个单位 → Kp=1 → output=3 → 向左修正
修到偏 1 个单位   → Kp=1 → output=1 → 修正力变小
修到偏 0 个单位   → 理想状态，但实际往往达不到
```

#### I（积分项）：`ki × integral`

- **作用**：累计过去的误差，消除静差
- **效果**：即使误差很小，积分也会随时间慢慢增大，直到把误差推回 0
- **风险**：积分**饱和**——如果车长时间无法归中（例如转弯），积分会累积巨大值，导致过冲

```c
// 积分饱和处理：全白脱线时清 0
if (全白) integral = 0;
```

#### D（微分项）：`kd × (error - last_error)`

- **作用**：预测误差的变化趋势，提前抑制过冲
- **效果**：误差在快速增大时提前加大修正，误差在快速减小时提前减小修正
- **类比**：看到车正在快速偏离，不等偏大就提前拉回来

```
当前 error = 3，上一次 error = 1
error - last_error = 2  → 误差在扩大 → D 项为正，加强修正

当前 error = 3，上一次 error = 5
error - last_error = -2 → 误差在缩小 → D 项为负，收小修正（防过冲）
```

### 6.4 循迹场景的三项作用形象化

| 场景 | P 作用 | I 作用 | D 作用 |
|---|---|---|---|
| 车在直道上轻微偏右 | 立即左转修正 | 微调消除残余偏差 | 抑制修正时的摆动 |
| 车进入右急弯 | 大误差 → 大转向 | 积分为转弯提供持续转向力 | 提前预判入弯力度 |
| 车出弯回正 | 转向力随误差减小 | 减速退绕 | 抑制回正过冲 |
| 车脱线全白 | 无误差 → 输出 0 | 清 0 | 无作用 |

### 6.5 与 electrical_machinery 的增量式 PI 的区别

```
位置式 PID（line_follower）:
  output = Kp·e + Ki·Σe + Kd·Δe
  每次输出绝对值，直接就是"该打多少方向"

增量式 PI（electrical_machinery）:
  Δu = Kp·Δe + Ki·e
  ControlVelocity += Δu
  每次输出增量，累加后才是"该加多少油门"
```

| 特性 | 位置式 PID | 增量式 PI |
|---|---|---|
| 输出 | 最终控制量 | 控制量的增量 |
| 积分 | 手动累加 `integral += error` | 通过静态变量自动累积 |
| 积分饱和 | 需要手动处理 | 天然不易饱和 |
| D 项 | 容易实现 | 对噪声敏感 |
| 适用场景 | 转向控制（期望值=0） | 速度控制（期望值≠0） |

### 6.6 调参建议

**经验口诀：先 P 后 D 再加 I**

```c
float kp = 0.5f;   // 先给一个较小的比例系数
float ki = 0.0f;   // 先关掉积分
float kd = 0.0f;   // 先关掉微分
```

**调参步骤：**

1. **调 P**：从小开始慢慢增大，直到小车能沿着黑线走（但可能有摆动）
   - P 太小 → 转弯无力，容易脱线
   - P 太大 → 左右剧烈摆动（蛇形）

2. **加 D**：抑制 P 引起的摆动，让车走得更直
   - D 太小 → 摆动没消除
   - D 太大 → 对噪声敏感，转向僵硬

3. **加 I**：消除长弯中的残余偏差
   - I 太小 → 长弯中持续偏外
   - I 太大 → 积分饱和导致过冲

## 7. 脱线转弯处理

### 7.1 触发条件

```c
// line_follower.c:40-47
if (s1 && s2 && s3 && s4 && s5 && s6 && s7 && s8)   // 全白（全部为 1）
{
    out_num ++;          // 连续脱线计数 +1
    actual_speed = 0;    // 停车
    if(out_num >= 3)     // 连续 3 次脱线
        trun_flage = true;  // 进入转弯模式
    integral = 0;         // 清积分防饱和
    return 0;             // 输出 0，不转向
}
```

### 7.2 转弯逻辑

```c
// line_follower.c:143-150
void car_trun(int8_t trun_speed)
{
    car_run(0, trun_speed);   // base=0 → 原地旋转
    if(!s4 || !s5)            // 中间传感器重新看到黑线（=0 = 黑线）
    {
        trun_flage = false;   // 退出转弯模式
    }
}
```

## 8. 完整集成示例

### 8.1 main.c

```c
#include "ti_msp_dl_config.h"
#include "clock.h"
#include "interrupt.h"
#include "electrical_machinery.h"
#include "line_follower.h"

uint8_t base_speed = 50;       // 基础速度 0-100
float kp = 0.8f;               // 比例系数
float ki = 0.0f;               // 积分系数（先关掉）
float kd = 1.2f;               // 微分系数

int main(void)
{
    SYSCFG_DL_init();
    SysTick_Init();
    DL_GPIO_setPins(GPIO_MOTOR_STBY_PORT, GPIO_MOTOR_STBY_PIN);
    
    while(1)
    {
        CAR_CONTROL();          // 执行一次循迹控制
        mspm0_delay_ms(20);     // 等待 20ms（与编码器周期同步）
    }
}
```

### 8.2 注意事项

1. **`base_speed`、`kp`、`ki`、`kd` 必须定义在全局作用域**，因为 `line_follower.c` 直接引用它们
2. **控制周期 20ms** 与编码器采样周期相同，保证传感器读取与控制同步
3. **STBY 引脚必须拉高**，否则电机无输出
4. **首次使用建议 I=0**，等 P 和 D 调好后再加 I

## 附录 A：加权平均法替代模式匹配

如果模式匹配覆盖不全导致异常，可以改用加权平均法：

```c
// 加权平均法计算偏差（替代查表）
float calc_error(void)
{
    int weighted_sum = 0;
    int total = 0;
    
    // 每个传感器赋予一个权重：左边负，右边正
    // s1=-3.5, s2=-2.5, s3=-1.5, s4=-0.5
    // s5=+0.5, s6=+1.5, s7=+2.5, s8=+3.5
    float weights[8] = {-3.5, -2.5, -1.5, -0.5, 0.5, 1.5, 2.5, 3.5};
    uint8_t sensors[8] = {s1, s2, s3, s4, s5, s6, s7, s8};
    int on_line_count = 0;
    
    for (int i = 0; i < 8; i++) {
        if (sensors[i] == 0) {   // 检测到黑线
            weighted_sum += weights[i] * 100;  // 放大整数化
            total += 100;
            on_line_count++;
        }
    }
    
    if (on_line_count == 0) return 999;  // 全白，脱线
    return (float)weighted_sum / total;   // 返回加权平均
}
```

优点：
- 覆盖任意传感器组合
- 偏差值是连续的，不跳变
- 对部分遮挡更鲁棒

## 附录 B：PD vs PID 适用速查

| 赛道特点 | 推荐控制 | 原因 |
|---|---|---|
| 直线多，弯道缓 | P 或 PD | 响应快，无积分饱和风险 |
| 急弯多 | PD + 前馈 | D 项提前入弯，避免减速过多 |
| 长距离弧线 | PID | I 项消除弧线中的持续偏移 |
| 赛道反光不一 | PD | I 项会因反光差异累积错误积分 |
